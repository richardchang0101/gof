工厂模式是简单工厂模式的延伸，继承了简单工厂模式的优点，同时还弥补了简单工厂模式的不足。
优点：
    1.工厂方法创建客户所需要的产品，同时还向客户隐藏可具体创建实例化的细节，用户调用对应的工厂就能实现实例化。
    2.基于工厂和产品角色多态性是设计工厂方法模式的关键。它能够让工厂自主确定创建何种产品对象，而如何创建这个对象的细节完全封装在具体工厂内部
    3.引入新产品时无需修改原有的代码，只需要实现抽象工厂方法，和实现具体的产品类即可。

缺点：
    1.在添加新产品时需要编写新的具体的产品类，而且还需要提供与之对应的具体工厂类。复杂度成对增加，对编译运行带来额外开销。
    2.由于考虑到系统的可扩展性，需要引入抽象层，在客户端代码中均使用抽象层进行定义，增建了抽象性，降低了可读性。
